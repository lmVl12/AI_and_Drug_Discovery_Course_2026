# AI in Drug Discovery: QSAR Modeling 

This repository contains my coursework and project implementations for the **AI and Biotechnology / Bioinformatics** program. The focus of this module is on **Quantitative Structure-Activity Relationship (QSAR)** modeling, a critical computational method for drug discovery.

The projects are implemented as a series of integrated steps, from chemical data processing to predictive model building.
The core of this project is focused on the computational study of **FLT3 (Fms-like tyrosine kinase 3)**, a high-profile target in oncology, particularly for Acute Myeloid Leukemia (AML). 
By applying **QSAR modeling**, I aim to predict the biological activity of small molecules against FLT3.

## 🧪 Project Overview

The goal of these tasks is to build predictive models that can relate the chemical structure of molecules to their biological activity. These notebooks cover the full QSAR workflow:

1.  **Data Collection & Preprocessing** — Handling chemical datasets (SMILES strings).
2.  **Molecular Descriptor Calculation** — Computing physicochemical properties and fingerprints (e.g., Lipinski descriptors, fingerprints).
3.  **Exploratory Data Analysis (EDA)** — Visualizing chemical space and activity distributions.
4.  **Model Building & Validation** — Training Machine Learning models (Random Forest, SVM, or Deep Learning) to predict bioactivity.

## 🏗 Repository Structure

The repository is organized into Jupyter Notebooks, each representing a specific phase of the QSAR pipeline:

* **`notebooks/`** — Contains `.ipynb` files with documented source code and visualizations.
* **`data/`** — raw and processed chemical datasets used for training.
* **`results/`** — Performance metrics, regression plots, and model evaluation summaries.

## 🛠 Tech Stack

* **Language:** Python
* **Domain Libraries:** RDKit (for cheminformatics), Pandas, NumPy.
* **Machine Learning:** Scikit-learn, Matplotlib, Seaborn.
* **Environment:** Jupyter Notebooks.
